¡ --> separador de inicio (tabla o columna)
! --> separador de fin (tabla o columna)
¿ --> separador de inicio (tipo de columna)
? --> separador de fin (tipo de columna)

Elementos de la base de datos
_ --> tabla
- --> columna

__ --> tabla
_# --> tabla del sistema
-@ --> partition key / clustering key
-$ --> columna recuperada (antes de insertar valores en esta clase de columnas se debe validar que el valor ya exista en la columna de referencia)
-& --> columna poblada desde el sistema
-* --> columna regular (este literal solo se usa cuando para la desambiguación de tablas, ver el ejemplo de "¡__!validaciones¡-*!descripcion")

Tipos de columnas (solo aplica para el tipo -)
V --> VARCHAR
I --> INT
T --> TIMESTAMP
L --> LIST
M --> MAP
S --> SET




COLUMNAS INTERMEDIAS
- un filtro no puede ser columna de consulta
- si una llave primaria de una tabla es columna de consulta y despues de ella no hay mas columnas de consulta, entonces no puede ser columna intermedia

(CreacionDinamicaTablas)
1. POR AHORA LAS BASES DE DATOS SE DISEÑAN A MANO A TRAVÉS DE ESTAS ESTRUCTURAS
2. LAS TABLAS SE DEBEN CREAN DESDE ESTAS ESTRUCTURAS

(IntegridadDatos)
3. A PARTIR DE ESTAS ESTRUCTURAS SE DEBE CREAR UNA HERRAMIENTA ENCARGADA DE MANTENER LA INTEGRIDAD DE LOS DATOS

(AdministracionFormularios)
4. LOS FORMULARIOS Y SUS ESTRUCTURAS HTML O MOBILE YA SE CREARÁN DESPUÉS DE LOS EVENTOS
